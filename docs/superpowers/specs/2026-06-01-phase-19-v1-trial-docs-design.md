# Phase 19 V1 Trial Docs Design

Phase 19 consolidates V1 trial documentation so the current local build is easier to review, demo, and hand over for feedback.

## Goals

- Add one clear V1 trial release notes document.
- Summarize included modules, demo credentials, test routes, verification commands, and known limits.
- Link the release notes from README and existing V1 docs.
- Keep this phase documentation-only.

## Non-goals

- No code behavior changes.
- No UI changes.
- No database changes.
- No deployment changes.

## Acceptance Criteria

- A reviewer can open one document and understand how to trial V1 locally.
- Existing detailed docs remain linked.
- README points to the V1 trial release notes.
