# Phase 37 Dashboard Filter Context Design

## Goal

Make dashboard summary links easier to understand after they open filtered list pages.

## Scope

- Show a status filter summary on the Roster list when a status filter is active.
- Show a status filter summary on the Service Logs list when a status filter is active.
- Show a status filter summary on the Invoices list when a status filter is active.
- Provide a clear filter link back to the unfiltered list.
- Keep existing filters, results, and dashboard links unchanged.

## Example Text

- `Showing draft shifts.`
- `Showing approved service logs.`
- `Showing draft invoices.`

## Out Of Scope

- Changing filter behavior.
- Adding new dashboard metrics.
- Changing list table columns or actions.
