# Phase 41 Roster Filter Usability Design

## Goal

Make roster name search filters easier to understand for daily use.

## Scope

- Clarify Participant and Worker placeholders as keyword searches.
- Make roster filter summary text read as a search context.
- Preserve existing participant and worker name matching behavior.
- Preserve date and status filtering behavior.

## Text Changes

- Participant placeholder: `Participant name keyword`.
- Worker placeholder: `Worker name keyword`.
- Summary example: `Showing shifts matching participant "Ava" and worker "Wendy".`

## Out Of Scope

- Adding phone, email, or NDIS matching.
- Adding autocomplete.
- Changing roster search behavior or table layout.
